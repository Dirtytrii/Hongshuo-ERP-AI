# 红字冲销业务规则与验收标准

> 本文档为 P2-3 红字冲销功能的业务规则与验收标准模板。开发前需由业务/财务确认并定稿。

---

## 一、业务背景

- 财务/库存单据审批生效后，不允许简单「撤销」，否则会导致数据错乱（如材料已领用但库存回滚）。
- 采用**红字冲销**：发现错误时，新做一张**冲销单**（负数或冲销类型）抵消原单，保留全部操作痕迹，符合财务规范。

---

## 二、适用范围

| 单据类型 | 是否支持红字冲销 | 备注                               |
| -------- | ---------------- | ---------------------------------- |
| 财务支出 | 是               | 冲销后需回减项目成本               |
| 财务收入 | 是               | 冲销后需回减项目已收款             |
| 库存入库 | 是               | 冲销后需回减库存数量               |
| 库存出库 | 是               | 冲销后需回增库存、回减项目出库金额 |

---

## 三、业务规则

### 3.1 谁可发起冲销

- [ ] 仅管理员可冲销
- [ ] 原单创建人可冲销
- [ ] 原单审批人可冲销
- [ 1 ] 财务角色可冲销财务单，库管可冲销库存单

### 3.2 冲销单与原单的关联方式

- 冲销单必须关联**唯一原单**（原单 id）。
- 冲销单金额为**负数**，或使用「冲销」类型标识，系统汇总时与原单抵消。
- 原单状态：是否允许「已冲销」状态标记，防止重复冲销。

### 3.3 审批流程

- [ ] 冲销单无需审批，保存即生效
- [ 1 ] 冲销单需审批，流程与正常支出/收入/出入库一致
- [ ] 冲销单需更高权限审批（如仅管理员）

### 3.4 与项目成本/库存的回算规则

- **财务支出冲销**：冲销单审批通过后，从 `Project.materialCost` / `laborCost` / `otherCost` 中按原单金额回减。
- **财务收入冲销**：冲销单审批通过后，从 `Project.receivedAmount` 回减。
- **出库冲销**：冲销单生效后，回增库存、回减该项目出库汇总金额（若项目详情展示出库材料金额，需同步更新）。
- **入库冲销**：冲销单生效后，回减库存。

### 3.5 报表与展示

- 财务报表：冲销单是否单独列出，还是与正常单合并显示（正负抵消）。
- 项目成本：项目详情中成本是否区分「正常」与「冲销」或仅展示净值。
- 库存流水：冲销单是否与正常出入库同表展示并标注「冲销」。

## 四、验收标准（开发完成后核对）

| 编号   | 验收项       | 通过条件                                                                   |
| ------ | ------------ | -------------------------------------------------------------------------- |
| P2-3.1 | 财务支出冲销 | 可针对已审批支出创建冲销单；冲销单审批通过后，项目对应成本回减正确。       |
| P2-3.2 | 财务收入冲销 | 可针对已审批收入创建冲销单；冲销单审批通过后，项目已收款回减正确。         |
| P2-3.3 | 库存出库冲销 | 可针对已审批出库创建冲销单；生效后库存回增、项目出库金额回减。             |
| P2-3.4 | 库存入库冲销 | 可针对已审批入库创建冲销单；生效后库存回减。                               |
| P2-3.5 | 权限与状态   | 仅约定角色可发起冲销；原单不可重复冲销；冲销单状态与审批流程符合规则文档。 |
| P2-3.6 | 报表与追溯   | 报表/列表中可区分或筛选冲销单；原单与冲销单可关联查询。                    |

---

## 五、数据模型与接口建议（供开发参考）

- **财务**：`FinanceRecord` 增加字段如 `reversalOfId`（原单 id）、`isReversal`（是否冲销单）；或单独表 `FinanceReversal` 关联原单。
- **库存**：`StockLog` 增加 `reversalOfId`、`isReversal`；冲销单数量为负或类型为「冲销」。
- **接口**：`POST /api/finance/reversal`、`POST /api/stock/reversal` 或于现有 create 接口通过 type=reversal + originalId 区分。
