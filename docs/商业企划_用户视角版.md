# 商业企划（用户视角版）

本文档面向老板、管理层、业务负责人和潜在使用方，用“用户买不买账”的视角说明这套系统为什么值得上、适合谁、能解决什么问题，以及当前已经交付到了什么程度。

## 1. 一句话定位

宏硕建设 ERP 不是做“大而全”的通用 ERP，而是面向工程类企业，把“项目经营、材料流转、财务收支、审批协同、经营预警”这几条最关键的链路串成一个轻量可落地的管理平台。

## 2. 用户最真实的痛点

建筑与工程类企业最常见的问题，不是没有数据，而是数据分散、口径不一致、责任不清楚。

管理层常见痛点：

- 项目赚不赚钱，要等月底甚至等结算才知道
- 该催的款没有及时催，现金流被动
- 项目超支了很久，系统和表格都没有第一时间提醒
- 供应商应付、已付、欠款分散在不同台账
- 审批和业务录入脱节，单据批了但经营台账没跟上

一线人员常见痛点：

- 项目经理看不到统一口径的项目成本和回款进度
- 财务录了支出，但项目侧不知道这笔钱算到哪里
- 材料出库和项目使用对不上，月底核对非常痛苦
- 多系统、多表格切来切去，录一笔数据要重复做很多次

## 3. 这套系统解决问题的方式

### 3.1 以项目为中心

项目是所有经营数据的主轴。合同、回款、材料、成本、预算、里程碑、变更、资料，都围绕项目归集。

### 3.2 以经营结果为导向

系统不是只记录“发生了什么”，而是持续回答几个管理上最关心的问题：

- 这个项目现在赚不赚钱
- 哪些钱应该催收但还没回
- 哪些项目已经接近或突破预算
- 哪些里程碑超期，背后可能带来结算和回款风险
- 哪些材料即将短缺，可能影响施工和成本

### 3.3 以角色分工推动落地

不同角色只看到自己该看、该管、该处理的内容：

- 老板和管理层看经营结果和风险
- 项目经理看自己负责的项目
- 财务看收支、报销、借还款和审批
- 仓库和录入岗位维护材料和出入库动作
- 管理员做权限、日志和集成配置

## 4. 从用户角度看，系统带来的价值

### 对老板 / 管理层

- 每天打开系统就能看到经营预警，不再只等月底报表
- 回款、预算、进度、库存等风险集中可见
- 对项目责任划分更清楚，便于管理追踪

### 对项目经理

- 只看自己负责的项目，信息更聚焦
- 能直接看到项目成本、利润、回款节点和超期里程碑
- 变更/签证、资料归集和项目执行记录更方便对账和复盘

### 对财务

- 收入、支出、报销、借还款和审批逻辑收口
- 审批通过后，项目经营口径能更快对齐
- 报表、应收应付、项目利润的核对成本更低

### 对仓库 / 录入岗位

- 入库、出库、库存预警有统一入口
- 模板下载和批量导入更规范
- 材料与项目、供应商关系更清楚

## 5. 当前已能交付的业务能力

以当前仓库和代码实现为准，Phase 1 到 Phase 4 的核心能力已经落地，可支撑正式演示、内部使用和继续迭代：

- 项目、合同、里程碑、回款计划、经营看板
- 预算执行看板与预算预警
- 报销、借款、还款业务链路
- 部门主数据与部门维度归集
- 审批中心统一待办
- 集成中心、移动端相关接口
- 资料库自动归集
- 用户、角色、权限、日志等基础能力

当前正在进行的是 Phase 5：

- 应用壳层重构
- 消息中心和预警聚合收口
- 共享页面骨架统一
- 文档、结构和可维护性治理

这意味着当前工作重点是“让系统更稳、更易交接、更易维护”，不是重新做一套新功能。

## 6. 适合什么类型的企业

这套系统更适合以下场景：

- 工程类、施工类、项目制交付企业
- 当前主要靠 Excel + 微信 + 人工对账维持运转
- 业务规模不算特别大，但项目多、链路复杂、管理痛点明显
- 需要先做轻量数字化收口，而不是上重型 ERP

不适合的场景：

- 强制造业、强生产排产场景
- 需要复杂供应链、多工厂、多账套的超大型集团化场景

## 7. 从实施角度的建议

建议不要一上来就追求“所有模块一次性全部上线”，而是按业务价值排序推进：

1. 先把项目、财务、材料三条主线跑通
2. 再让审批中心、消息中心、集成中心参与协同
3. 最后再补深度报表、AI 分析和更细的流程治理

这种节奏的好处是：

- 前期见效快
- 用户更容易接受
- 数据质量更容易逐步稳定

## 8. 对外介绍时可以怎么说

如果需要面向客户、老板或合作方做一页式介绍，可以直接用下面这段口径：

> 宏硕建设 ERP 是一套面向工程类企业的轻量经营管理平台，围绕项目、材料、财务和审批协同展开，重点解决“项目盈亏不清、回款跟踪滞后、成本口径不统一、风险提醒不及时”等问题。系统已经具备项目管理、合同管理、报销借还款、预算预警、审批中心、集成中心等核心能力，当前正在进行结构和交付层优化，适合中小型工程企业分阶段上线。
