# 宏硕建设 ERP - 二期开发计划

> 本文档供 Web Agent 读取并执行开发。对照钉钉「工程通项目管理」成熟产品，补齐合同全生命周期、经营/预算看板、员工报销与借还款、部门成本、审批流与看板穿透等能力。按 Phase 1 → Phase 4 顺序实施，每阶段交付物见文末。

## 一、工程通（钉钉）核心能力摘要

依据 [钉钉工程通方案页](https://www.aliwork.com/o/lca/solution?spm=a3140.8196062.0.0.532f2404Ergb3d&id=56&channel=DDguanwang) 介绍，工程通主打：

- **主线**：项目为主线，收入与支出为抓手，**合同、成本、进度**为基石；项目现场人、机、料、法、环。
- **一体化**：合同签订、结算、监控各阶段流程数据集成，**项目一体化视图**，实时更新。
- **看板**：**经营看板、预算执行看板**自动生成，合同签订/结算/收支/员工报账等业务执行数据自动统计。
- **风险**：集成九大数据，披露关键数据，**分析呈现运营风险**。
- **收支归集**：**自动归集项目收支、部门成本**，支持成本节约与资源优化。
- **审批**：**线上审批流转**，报销、借还款等待办统一集成至钉钉。
- **资料库**：自动归集项目相关业务流形成**线上资料库**；移动化办公、钉钉待办/通知。
- **穿透**：按收支线与项目统计设置**单击穿透**，进入对应执行表单与报表。

### 当前落地进度（更新于 2026-03-14）

- ✅ **Phase 1 完成**：合同全生命周期、经营看板、预算执行看板、仪表盘穿透与运营风险区块。
- ✅ **Phase 2 完成**：报销单、借款单、还款单 CRUD + 状态流（draft/submitted/approved/rejected）+ 审批动作。
- ✅ **Phase 3 完成（核心）**：部门主数据、部门成本归集、用户/财务部门维度。
- ✅ **Phase 4 完成（本次补充）**：
  - 资料库自动归集（contract/change_order/reimbursement/finance/loan/loan_repayment）；
  - 集成中心（钉钉 webhook 配置、测试发送、移动端接口开关）；
  - 轻量移动端接口：`/api/mobile/overview`、`/api/mobile/todos`；
  - 审批中心聚合待办接口：`/api/approval-center/todos`。

## 二、功能差异对比表

| 能力项              | 工程通                              | 宏硕 ERP 现状                                        | 差距                                 |
| ------------------- | ----------------------------------- | ---------------------------------------------------- | ------------------------------------ |
| 项目+收支+成本+进度 | 有                                  | 有（项目、收支、成本、里程碑、回款计划）             | 基本对齐                             |
| 合同                | 签订、结算、监控全阶段              | 仅项目上 `contractAmount` 字段，无合同实体与阶段     | **缺合同全生命周期**                 |
| 经营看板            | 自动生成，合同/结算/收支/报账等统计 | 仪表盘有待催款、超期里程碑、预算预警                 | **缺「经营看板」命名与自动归集规则** |
| 预算执行看板        | 独立看板，预算 vs 执行              | 有预算与超支预警（80%/100%），无独立「预算执行看板」 | **缺预算执行看板视图**               |
| 运营风险/KPI        | 九大数据、风险披露                  | 有报表与仪表盘，无专门「运营风险」看板               | **可增强**                           |
| 部门成本            | 按部门集中归集                      | 无部门维度                                           | **缺部门与部门成本归集**             |
| 员工报销            | 有，并播报                          | 无                                                   | **缺报销单**                         |
| 借还款              | 有，并播报                          | 无                                                   | **缺借款/还款单**                    |
| 审批流转            | 通用线上审批、待办集成              | 仅财务审批、变更单审批（单点审批）                   | **缺通用审批流（多级/会签/转办）**   |
| 项目资料库          | 业务流自动归集为资料库              | 有 ProjectDocument（名称+链接+备注）                 | **缺「自动归集业务流」形成资料库**   |
| 流程穿透            | 单击进表单/报表                     | 部分有（如项目详情），仪表盘穿透不足                 | **可增强仪表盘穿透**                 |
| 钉钉/移动           | 钉钉待办、通知、移动审批            | 无                                                   | **缺钉钉集成与移动端**               |

当前业务开发进度见 `docs/DEVELOPMENT_PLAN_BUSINESS.md`：P0/P1 已完成，P2 部分完成（里程碑超期预警、红字冲销已做；甘特图、图纸管理待排期）。

### Phase 1：合同与看板（约 3–4 周）

#### 1.1 合同全生命周期

- **后端**
  - 新增实体 `Contract`（表名 `contracts`）：id、projectId、合同金额、签订日、结算状态（未结算/部分结算/已结算）、监控状态、创建/更新时间等；与现有 `Project.contractAmount`、变更单回写逻辑兼容（合同金额可继续由项目+变更单汇总得出，或由合同主数据驱动项目合同额）。
  - 新增 `ContractRepository`、`ContractService`、`ContractController`，提供 CRUD 及按项目、按结算/监控状态筛选。
  - 可选：合同与回款计划节点绑定，便于「按合同看回款」。
- **前端**
  - 合同列表/筛选页（按签订、结算、监控阶段）；与回款计划、收入关联展示；项目详情或项目列表可进入合同列表。

#### 1.2 经营看板

- **后端**
  - 新增聚合接口（如 `GET /api/dashboard/operation`）：统计合同签订额、结算额、项目收支、待催款金额、超支项目数等；复用现有 `ProjectController`、`FinanceController`、`PaymentPlanController` 等数据。
- **前端**
  - 在现有 `components/Dashboard/Dashboard.tsx` 上增加「经营看板」区块（或独立 Tab）：展示上述指标，命名与归集规则与接口一致。

#### 1.3 预算执行看板

- **后端**
  - 新增聚合/筛选接口（如 `GET /api/dashboard/budget-execution`）：按项目（后续可扩展按部门）返回预算、实际成本、执行率、预警状态；与现有 `Project` 的 `totalBudget`、`actualCostTotal`、`budgetRatio`、`budgetAlertStatus` 一致。
- **前端**
  - 新增「预算执行看板」页或 Dashboard 区块：表格/卡片展示「预算 vs 实际成本 vs 执行率」，支持点击穿透到项目详情。

#### 1.4 仪表盘穿透

- **前端**
  - 待催款、超期里程碑、预算预警等卡片/列表：点击条目跳转到项目详情、回款计划、财务明细或报表（路由与参数对接现有页面）。
  - 可选：新增「运营风险」小区块（如超支项目数、逾期回款笔数），并支持穿透到明细。

---

### Phase 2：员工报销与借还款（约 2–3 周）

#### 2.1 员工报销单

- **后端**
  - 新增实体 `Reimbursement`（表名 `reimbursements`）：id、projectId、departmentId（可选，Phase 3 再必填）、报销人（userId/username）、金额、类别、状态（draft/submitted/approved/rejected）、创建/更新/审批人/审批时间等。
  - 新增 `ReimbursementRepository`、`ReimbursementService`、`ReimbursementController`；状态流：草稿 → 提交 → 审批中 → 已通过/已拒绝。
  - 审批：复用或扩展现有财务审批逻辑；审批通过后可选择「回写项目成本」或仅作费用台账（与 `FinanceRecord` 或项目成本字段的联动规则需明确）。
- **前端**
  - 报销单列表/筛选、新建/编辑/提交/审批；与经营看板联动（如「本月报销金额」统计）。

#### 2.2 借还款单

- **后端**
  - 新增实体 `Loan`（借款单，表名 `loans`）：id、借款人、金额、项目/部门、事由、状态等。
  - 新增实体 `LoanRepayment`（还款单，表名 `loan_repayments`）：id、loanId、还款金额、状态、还款时间等。
  - 新增 Repository/Service/Controller；审批与状态流同报销；可选与报销单联动（报销时勾选「冲抵借款」）。
- **前端**
  - 借款单/还款单 CRUD、列表、审批；仪表盘/经营看板增加「近期借还款/报销」列表或数字摘要。

---

### Phase 3：部门与审批流（约 3–4 周）

#### 3.1 部门与部门成本

- **后端**
  - 新增实体 `Department`（表名 `departments`）：id、name、code、parentId 等；用户表增加 departmentId；项目可关联部门或通过项目经理所在部门推导。
  - 成本归集：支出/报销/借还款按部门汇总；新增「部门成本」报表接口或看板数据接口。
- **前端**
  - 部门主数据维护（列表/树、增删改）；部门成本报表或看板视图；与现有项目成本口径并存。

#### 3.2 审批流增强

- **后端**
  - 当前仅为「单笔审批」：`FinanceController`、`ChangeOrderController` 的审批接口。
  - 目标：支持多级审批、可选会签/或签、转办；审批流配置（按单据类型、金额阈值等）；待办列表接口（我发起的、待我审批的）。
  - 可与报销、借还款、变更单、财务支出统一为「审批中心」入口，为后续钉钉待办打基础。
- **前端**
  - 审批流配置页；统一「审批中心」入口（待办列表、处理操作）；各业务单据入口保留现有入口并可与审批中心联动。

---

### Phase 4：资料库与集成（约 2–4 周，视是否做钉钉而定）

#### 4.1 资料库自动归集

- **后端**
  - 在现有 `ProjectDocument`（`src/main/java/com/hongshuo/erp/model/ProjectDocument.java`）上增加「来源」字段（如 contract、change_order、reimbursement、finance 等）；关键业务动作（如合同签订、变更单审批通过、报销审批通过）自动写入一条资料库记录（标题+链接到业务单+时间）。
- **前端**
  - `components/ProjectDetail/ProjectDocumentList.tsx` 支持按来源筛选与展示；项目详情资料库列表展示自动归集条目。

#### 4.2 钉钉/移动端（可选）

- 钉钉工作台/待办：审批待办、催款提醒等推送钉钉；需钉钉开放平台应用、OAuth 与 API 对接。
- 或先做轻量移动端：现有 `miniprogram/` 或 H5 做「待办+审批+关键看板」，与后端现有 API 对接，不改动后端核心逻辑。
